## 最接近的竞争对手

### eMEM（Automatika Robotics）—— 直接竞争者
- **优势：** 先发优势，具备真正的空间记忆，可运行于 Jetson，已与其 EMOS 技术栈集成。
- **劣势：** 与 ROS 2 强绑定（Python 核心，但所有示例均假设存在 ROS topics）；无开放词汇语义（封闭标签集）；仅 2 ★ / 小众品牌；许可证尚可但无社区。
- **我们的优势：** 框架无关的核心（ROS 2 是适配器，而非依赖）。从第一天起就采用 CLIP/SigLIP 驱动的开放词汇。无需安装 ROS 的更快 `add_frame` → `query` 路径。更好的开发者体验。

### Mem0 —— 相邻领域，非竞争对手
- **定位：** 盟友 / 灵感来源。他们专注文本记忆，我们形成互补。
- **策略：** 提供一个 `mem0` 适配器，让使用 Mem0 的 agent 能够将 SpatialMem 接入作为"空间后端"，从而实现跨模态的统一记忆。具备联合营销潜力。

### ConceptGraphs / Hydra / Open3DSG / DovSG —— 上游
- **定位：** 这些是可从我们摄入层内部调用的*感知后端*，而非竞争者——它们是供应商。
- **策略：** 选取 **ConceptGraphs** 作为 v0 融合后端（Apache-2.0 协议，支持 RGB-D，引用量高，演示友好）。后续再添加 Hydra 和 nvblox-feature 适配器。

### NVIDIA Isaac / GR00T / Cosmos —— 相邻平台
- **定位：** 他们提供感知 + VLA 大脑，但明确不提供持久化空间记忆。
- **策略：** 发布 Isaac ROS / GR00T 集成方案。成为"GR00T 遗忘的记忆层"。
- **验证（2026-06）：** NVIDIA 官方 **Cosmos 3** 技术报告（138 页，GTC Taipei）将此空白明确定义为该领域的开放问题——*"物理智能所需的不仅是识别……它需要时间持久的状态、与对象和 agent 绑定的空间定位……一个可持续维护、可执行的场景估计。"* 而 Cosmos 3 本身并未提供任何此类能力：仅有一个上限为 74K token 的生成上下文、每次请求独立的临时 KV-cache，通篇不提"空间记忆"或"跨会话"。这款旗舰 Physical-AI 模型**将我们的核心论点定义为尚未解决的问题**——这是对该切入点最有力的第三方背书。
- **接缝，而非威胁：** Cosmos 3 的 Reasoner 输出*结构化的相机坐标系 3D 框（统一 JSON）+ 估算的度量自我姿态*——即它是我们摄入契约（`Cosmos3PerceptionAdapter`）的强力**上游检测来源**，而非记忆竞争者。我们构建于其上，持久化它所感知的一切。

## 三句话定位（请熟记）

> Mem0 破解了 agent 的文本记忆。NVIDIA 破解了具身 AI 的感知。两者之间的空间记忆无人破解——每个机器人实验室都在自行重建。SpatialMem 是开放的、可 pip 安装的、框架无关的空间记忆层，将 RGB-D 流转化为可持久化、可被 LLM 查询的场景图。

## 五年威胁模型

| 风险 | 可能性 | 应对措施 |
|---|---|---|
| Mem0 推出 3D 扩展 | 中 | 快速推进，建立品牌，寻求收购要约或保持友好合作伙伴关系 |
| NVIDIA 将其纳入 Isaac | 中 | 保持框架无关；他们的技术栈依赖 CUDA，我们可在 Mac/CPU/Jetson 上运行 |
| eMEM 获得牵引力 | 低 | 其 ROS 绑定是非机器人用户的永久天花板 |
| 新的学术 SOTA 场景图涌现 | 高 | 我们是*层*，而非后端——通过适配器将其换入即可 |
| LLM 获得世界模型，使显式记忆变得多余 | 中期 | 对我们有利：记忆 token 仍然需要来自某个持久化的地方 |
