# 00 · 愿景

> 单包（记忆核）愿景。系统级（SpatialRobot 三层整体）愿景见 [../../../docs/zh/VISION.md](../../../docs/zh/VISION.md)。

## 一句话定位

**SpatialMem 是持久化、可查询、面向 LLM 的空间记忆层，位于机器人/AR/智能体的感知栈与语言模型之间——Mem0 之于文本，SpatialMem 之于三维空间。**

## 问题

当前，具身或 AR 智能体通常由三个彼此割裂的部分组成：

1. **感知**（SLAM + VLM）：产生瞬态点云、占用栅格、逐帧检测结果。
2. **推理**（LLM / VLA）：消费*当前*感知输出，选择动作。
3. **长期记忆**：不存在。或者是自制的场景图 pickle，既不能在重启后恢复，也不支持增量更新，更无法用自然语言查询。

每个具身团队都在重建同一套管道——融合检测结果、去重对象、持久化、为 LLM 序列化。但因为这不是其核心 IP，所以每家都做得很糟糕。

## 我们构建什么

一个**框架无关的空间记忆 SDK**，包含三个契约：

- **摄取契约** — `add_frame(rgb, depth, pose)` 或 `add_pointcloud(...)` 或 `add_detections(...)`。可替换的感知后端（初期采用 ConceptGraphs 风格的 RGB-D 融合；后期支持 Hydra、nvblox-features、自定义后端）。
- **记忆契约** — 增量式场景图（对象、地点、房间、关系），持久化到单文件存储。支持重启恢复，支持 update / forget / decay。
- **查询契约** — `mem.query("where is the red mug?")` 返回有根据的答案及节点 id；`mem.serialize()` 将压缩后的图文本导出供任意 LLM prompt 使用；`mem.spatial(near=..., radius=...)` 支持几何过滤。

## 非目标

- **不是新的 SLAM。** 位姿来自上游（ORB-SLAM3、cuVSLAM、ARKit、仿真真值）。
- **不是新的 VLM。** 嵌入向量来自上游（CLIP / SigLIP / 用户自有模型）。
- **不是机器人操作系统。** 无运动规划，无 ROS 硬依赖。ROS 2 节点作为适配器提供，而非核心。
- **不优先做云服务。** 本地优先、嵌入式部署、BYO-LLM。托管层作为便利性选项在后续推出，而非护城河。

## 切入点

> Mem0 是文本记忆，A 轮融资 2400 万美元，GitHub 56k★。空间记忆是同一类问题，但基底更大，目前**一片空白**。

目前最接近的产品是 `eMEM`（锁定 ROS 2、2★、无开放词汇语义）。学术仓库（ConceptGraphs、Hydra、Open3DSG、DovSG）全都止步于"图已构建"——没有任何一个将持久化、查询或 LLM 序列化产品化。

相邻仓库中的 L3 规划器 / Brain2Robot 工作是内置的旗舰消费方——我们交付规划器所需的记忆层，并附上真实的"记忆→规划"演示，这是任何学术仓库都无法匹敌的。

## 目标受众（优先级排序）

1. **具身智能体研究者**，需要开箱即用的空间记忆，不想重新实现对象去重和持久化。以 `pip install` + 10 行示例代码赢得他们。
2. **人形机器人 / 移动机器人初创公司**，需要多日连续自主运行能力。以 ROS 2 适配器 + 漂移条件下的持久化基准赢得他们。
3. **AR/XR 应用开发者**（Vision Pro、Quest），需要跨会话场景记忆。后续以 iOS/Android 桥接赢得他们。
4. **智能家居 / 设施机器人。** 同类问题，后续推进。

## 押注

2026 年的两大叙事——"智能体记忆"（Mem0 论点）与"Physical AI"（NVIDIA 论点）——正在收敛。SpatialMem 是唯一有望在两个故事中同时被引用的库。我们胜出的关键在于交付最简洁的 API，而非最深厚的研究。

NVIDIA 2026 年 6 月发布的 **Cosmos 3** 技术报告为我们的押注背书：报告将*"时间持久状态、与对象和智能体绑定的空间定位……一个可维护、可操作的场景估计"*定义为 Physical AI 尚未解决的挑战——但其发布的仍是一个具有有界上下文且**没有**持久场景存储的逐片段模型。Physical AI 领域最大的玩家刚刚将我们的这一层定义为缺失的拼图。Cosmos 3 的 Reasoner（结构化相机帧 3D 框 + 度量自车位姿）将成为我们摄取的*上游数据源*，而非竞争对手——它负责感知，我们负责记忆。
