# 05 · 开放性问题

尚未锁定的实时设计决策。每项均有负责人决策截止日期（即必须解决的里程碑节点，否则将成为阻塞项）。已解决的条目移至底部的带日期决策日志。

## 融合

| # | 问题 | 重要性 | 决策截止 |
|---|---|---|---|
| F1 | 逐帧同步融合还是提交时批量融合？ | 延迟与吞吐量之间的权衡；影响 `add_*` 返回值的 API 契约 | M1 |
| F2 | 静态与动态对象处理——杯子会移动，冰箱不会 | 错误处理 = 要么产生幽灵重复（将移动的杯子视为新对象），要么位置过时 | M2 |
| F3 | 是否支持按类别覆盖阈值（每个标签单独设置 `τ_merge`）？ | 全局阈值会错误合并小物体；按类别配置需要一个配置界面 | M2 |
| F4 | 节点积累数百次观测时，如何保持特征质心的稳定性？ | EMA 漂移可能将节点的嵌入向量拉离其真实身份 | M2 |
| F5 | M2 时分割检测默认开启还是关闭？ | 关闭 = 斑块合并；开启 = 计算开销 + 误分割 | M2 |

## Schema / 存储

| # | 问题 | 重要性 | 决策截止 |
|---|---|---|---|
| S1 | 单个 `.smem` 文件还是每个 episode 一个文件？ | 多会话机器人可能需要 episode 隔离 + 低成本剪枝 | M3 |
| S2 | 嵌入维度在 `open` 时锁定——若用户从 CLIP 切换到 SigLIP，迁移路径是什么？ | 对整个 store 重新嵌入代价高昂；需要重建索引工具 | M2 |
| S3 | 存储原始点云还是仅存储聚合后的几何信息？ | 原始 = 后续可再分割/重融合，但磁盘占用增加 10–100× | M1 |
| S4 | 超过约 100 万向量时，使用 sqlite-vec 还是 faiss-on-disk？ | sqlite-vec 在大规模场景下的召回率/延迟未经验证 | M3 |
